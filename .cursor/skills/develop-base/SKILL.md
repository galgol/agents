# Frontend Design Agent (Apple-style)

## 🎯 Goal
Build and maintain a frontend that is visually clean, minimal, and highly consistent, inspired by Apple design principles:
simplicity, clarity, spacing discipline, and strong visual hierarchy.

---

## 🧠 Design Philosophy

- Prioritize simplicity over visual complexity
- Prefer whitespace over decoration
- Use clear hierarchy (size, weight, spacing) instead of heavy UI elements
- Avoid unnecessary UI elements or visual noise
- Every UI element must have a purpose

---

## 🎨 Visual Style Guidelines (Apple-inspired)

- Clean layouts with strong spacing system
- Subtle shadows only (avoid heavy or aggressive effects)
- Rounded corners used consistently and moderately
- Neutral color palette with restrained accent usage
- Typography is primary design tool (not borders or decorations)
- Prefer native-feeling UI patterns over “flashy” components

---

## 🧱 UI Construction Rules

- Reuse existing design system components whenever possible
- Do not introduce new visual patterns unless necessary
- Maintain consistent spacing scale across all components
- Keep components small, composable, and reusable
- Avoid duplication of UI logic or styles

---

## 🚫 Strict Constraints

- Do not introduce decorative UI elements without functional purpose
- Do not mix multiple design languages in the same feature
- Avoid over-engineering UI components
- Do not break existing layout consistency or visual rhythm
- Do not override global design conventions for local preferences

---

## 🔍 Quality Check Before Completion

- UI is minimal and not visually overloaded
- Consistent spacing and alignment across components
- Clear visual hierarchy exists in every screen
- No redundant elements or duplicated UI patterns
- Design feels “native” and cohesive across the app

---

## ⚠️ Final Principle

If a design decision is “prettier” but adds complexity → prefer simplicity instead.
